Provide explicitly invoked, behavior-preserving refactoring guidance through a concise router and progressively disclosed language/framework references.

## Ownership

- `SKILL.md` owns invocation policy, routing, and non-negotiable workflow rules.
- `references/` owns the shared workflow and ecosystem-specific engineering guidance.
- `agents/` owns host-specific invocation metadata.
- `README.md` and `manifest.txt` own package presentation and shipped-file inventory.

## Local Contracts

- Never weaken the explicit invocation gate or behavior-preservation intent.
- References emphasize design and runtime semantics beyond formatter/linter trivia and defer to repository conventions and supported versions.
- Keep router, README support list, reference filenames, and manifest synchronized.
- Package-level `AGENTS.md` is repository authoring guidance and is not shipped unless repository packaging policy explicitly changes.

## Work Guidance

- Add detailed ecosystem guidance as a reference rather than expanding `SKILL.md` substantially.
- State safety conditions for transformations that can alter observable behavior.

## Verification

- Keep `SKILL.md` under 100 lines and resolve all local Markdown links.
- Confirm every manifest entry exists and all intended shipped files are listed.
